5grid 0.4.3

A simple and flexible grid-based HTML5+CSS3 framework.

What is 5grid?

5grid is a responsive HTML5+CSS3 design framework that features:

  • A powerful grid system with a simple, concise syntax and numerous advanced features.
  • Strong HTML5 and CSS3 underpinnings.
  • Responsive handling with support for desktop, tablet, and mobile displays.
  • A slick, highly customizable mobile UI.

5grid is licensed under the MIT license and the GPLv2. You can find the latest version here.

How it Works

5grid is comprised of two components: a grid system and a responsive handler.

The Grid System

Grid-based design is a two part design technique: the first part divides up a working canvas into a number of equally spaced, fixed-width vertical columns (usually 12 or 16, but almost any number works), which are then used as "scaffolding" to ensure page elements are consistently sized and aligned. The second part, and arguably the most beautiful, is the CSS grid system, which lets a designer easily translate the underlying layout of their grid-based design into completely functional HTML and CSS. This gives designers the freedom to put together all kinds of complex layouts without having to worry about the CSS needed to get it working.

5grid is, as its name implies, a grid-based design framework, and at its heart is its own CSS grid system. Here's how it's set up:

  • Grids are divided up into 12 vertical columns of equal width, with gutters (also of equal width) separating each one.
  • Each column represents a single unit of measurement.
  • Grid elements can be anywhere from 1 unit wide (1u) up to 12 units wide (12u).
  • Grid elements can be right-shifted by any number of units using offsets (more on that here).
  • Each grid row can have any number of grid elements, but the sum of their units (offsets included) must be less than or equal to 12.
  • A grid can be either a layout grid, which is used to dictate the overall layout of a page, or a regular grid, which can be nested and used anywhere.

So what makes 5grid's system unique? Two things: its super-minimal syntax (which you'll see down in Basic Usage), and the fact that it's designed to play nice with 5grid's other (very important) component ...

The Responsive Handler

We're now at a point where folks do a good bit of browsing on small display devices like phones and tablets, but at the same time use desktops and notebooks with increasingly wider displays. Responsive design is a technique that automatically serves up your content with a stylesheet best suited for the display on which it's being viewed. When done right, the end result is a design that can transparently adapt to all kinds of devices and display resolutions while providing an optimal viewing experience across the board.

5grid includes a responsive handler to do just that: it first loads style.css (the global stylesheet), and then decides which supported display mode (listed below) is best suited to the available screen width. Once it's figured that out, it automatically loads the appropriate mode-specific stylesheet (in parentheses).

1200px (style-1200px.css)
Targets widescreen desktops/notebooks. Layout grid widths are fixed at 1200px, placing them safely below the most popular widescreen resolution (1366x768) while still giving them plenty of room to breathe.
1000px (style-1000px.css)
Targets tablets and lower-resolution desktops/notebooks. Layout grid widths are fixed at 1000px, which is optimal for almost all tablets (10" and smaller) while still safely below the resolution that just doesn't seem to die (1024x768).
desktop (style-desktop.css)
Targets all desktop sizes. Handy if you've already designed things in such a way that you don't need completely separate stylesheets for 1000px and 1200px. It can, however, be used in conjunction with 1000px and/or 1200px to load additional stylesheets targeting each of those specific modes.
mobile (style-mobile.css)
Targets mobile devices (primarily phones). This mode differs from the others in that each grid element is forced to its own row (which sounds a bit weird at first, but the result is a significantly better experience on small/narrow displays). All grid widths are also fluid to elegantly handle both portrait/landscape viewing and the ridiculous number of screen resolutions found on mobile devices. As of version 0.3, this mode also has an optional and very customizable mobile UI with a fixed titlebar and a cool slide-out navigation panel (more on that here).
fluid (style-fluid.css)
This is a special desktop/notebook mode that sets layout grid widths to fluid, which is handy if you need to go with a truly fluid layout. Note that this mode can only be used on its own or in conjunction with mobile.

Note: While the responsive handler is really cool, designing a properly responsive design does take a bit more work due to the additional stylesheets. If that's not something you want to get into just yet, you do have the option to initially skip the responsive handler entirely (see Getting Started).

Getting Started

It's real simple:

  1. Make a copy of the template/ folder and rename it to something else. This will be your work folder.

  2. (Optional) Override 5grid's default settings. You can do this by either directly editing the first section of 5grid/init.js or by passing custom settings to its <script> tag in standard query format. For example: 

    <script src="5grid/init.js?use=mobile,1200px&mobileUI=1"></script>

  3. You're done – onward to Basic Usage!

Basic Usage

Creating a grid is really straightforward. Here's the basic syntax:

<div class="5grid">
  <!-- Row 1 -->
    <div class="row">
      <div class="[width]u">...</div>
      <div class="[width]u">...</div>
      <div class="[width]u">...</div>
    </div>
    ...
  <!-- Row 2 -->
    <div class="row">
      <div class="[width]u">...</div>
      <div class="[width]u">...</div>
      <div class="[width]u">...</div>
    </div>
  ...
</div>

Which breaks down like this:

<div class="5grid">
Starts a new grid. If this grid is going to be a layout grid (ie. you'll be using it for your page layout), you'll want to use <div class="5grid-layout"> instead.
<div class="row">
Starts a new grid row.
<div class="[width]u">...</div>
Defines a grid element of [width] units (between 1 and 12). Each row can have any number of these provided the sum of their units are less than or equal to 12.

</div>

Advanced Usage

5grid's grid system was designed to be simple and lightweight, but it's still capable of handling some pretty sophisticated (and sometimes unusual) use cases:

Nesting

Let's say you have the following grid:

Code:
<div class="5grid-layout">
  <div class="row">
    <div class="9u">9u</div>
    <div class="3u">3u</div>
  </div>
</div>
Output:
9u
3u

And you decide you want to partition the 9u element into three 3u elements, but as children of the 9u element. You can do this with nesting, which allows you to place regular grids inside any grid element. In this case, we'll need to place a grid with a row of three 4u elements to get the desired effect. Here's the result:

Code:
<div class="5grid-layout">
  <div class="row">
    <div class="9u">
      <div class="5grid">
        <div class="row">
          <div class="4u">4u</div>
          <div class="4u">4u</div>
          <div class="4u">4u</div>
        </div>
      </div>
    </div>
    <div class="3u">3u</div>
  </div>
</div>
Output:
4u
4u
4u
3u

Offsets

Offsets let you right-shift grid elements by any number of grid units, like so:

Code:
<div class="5grid-layout">
  <div class="row">
    <div class="3u offset-9u">4u (offset by 9)</div>
  </div>
  <div class="row">
    <div class="3u offset-5u">3u (offset by 5)</div>
    <div class="3u offset-1u">3u (offset by 1)</div>
  </div>
  <div class="row">
    <div class="3u offset-1u">3u (offset by 1)</div>
    <div class="3u offset-1u">3u (offset by 1)</div>
    <div class="3u offset-1u">3u (offset by 1)</div>
  </div>
</div>
Output:
4u (offset by 9)
3u (offset by 5)
3u (offset by 1)

NoScript

5grid needs JS to do its thing, so if you're expecting some of your users to have it disabled add the following to your <head> container to manually load in your stylesheets in those situations:

<noscript>
  <link rel="stylesheet" href="5grid/core.css" />
  <link rel="stylesheet" href="5grid/core-desktop.css" />
  <link rel="stylesheet" href="5grid/core-(1000px or 1200px).css" />
  <link rel="stylesheet" href="5grid/core-noscript.css" />
  <link rel="stylesheet" href="style.css" />
  ... (link tags to your desktop, 1000px, and/or 1200px stylesheets) ...
</noscript>

Note that the responsive handler won't work if JS is disabled, so you'll need to select the mode to use (desktop 1000px or desktop 1200px) by including the appropriate stylesheets. For example, the following will make NoScript users default to the desktop 1200px mode (relevant lines are in bold):

<noscript>
  <link rel="stylesheet" href="5grid/core.css" />
  <link rel="stylesheet" href="5grid/core-desktop.css" />
  <link rel="stylesheet" href="5grid/core-1200px.css" />
  <link rel="stylesheet" href="5grid/core-noscript.css" />
  <link rel="stylesheet" href="style.css" />
  <link rel="stylesheet" href="style-desktop.css" />
</noscript>

Flush Grids

To create a flush (gutterless) grid, simply add the 5grid-flush modifier class to your grid:

Code:
<div class="5grid 5grid-flush">
  <!-- Row 1 -->
    <div class="row">
      <div class="3u">3u</div>
      <div class="6u">6u</div>
      <div class="3u">3u</div>
    </div>
  <!-- Row 2 -->
    <div class="row">
      <div class="4u">4u</div>
      <div class="4u">4u</div>
      <div class="4u">4u</div>
    </div>
</div>

Mobile UI

What is it?

The mobile UI is an optional, customizable interface that makes getting around your site much easier on mobile devices. It's made up of two components:

Title Bar
A horizontal bar fixed at the top of the screen with your site's name and a button to activate the ...
Navigation panel
A navigation panel that slides out of the left side of the screen when activated.

Enabling the mobile UI

  1. Set mobileUI to 1.

  2. Add the class mobileUI-site-name to the element containing your site's name (so 5grid knows where to find it). For example:

    <h1 class="mobileUI-site-name">Placeholder</h1>

  3. Similar deal here: add the class mobileUI-site-nav to the element with your site's main navigation links. For example:

    <nav class="mobileUI-site-nav">
  <a href="#">Home</a>
  <a href="#">Services</a>
  <a href="#">Portfolio</a>
  <a href="#">About</a>
  <a href="#">Contact</a>
</nav>

    Note that your main navigation can be in any structure, even multi-level ones like this:

    <nav class="mobileUI-site-nav">
  <ul>
    <li class="current_page_item"><a href="#">Home</a></li>
    <li>
      <a href="#">Services</a>
      <ul>
        <li><a href="#">Service 1</a></li>
        <li><a href="#">Service 2</a></li>
        <li><a href="#">Service 3</a></li>
      </ul>
    </li>
    <li><a href="#">Portfolio</a></li>
    <li><a href="#">About</a></li>
    <li><a href="#">Contact</a></li>
  </ul>
</nav>

And that's it! Here's how it'll look (using the default Modern theme and the default color palette):

You can also tweak the look and behavior of the mobile UI using a number of settings.

Classes

There are a handful of classes you can assign to your page elements to give them special meaning to the mobile UI:

mobileUI-site-name
Marks the element containing your site name (obtained with jQuery.html()).
mobileUI-site-nav
Marks the element containing your main navigation links. The mobile UI will copy all anchor elements it finds in this element to the navigation panel.
mobileUI-main-content
Marks the parent element of your main content. If used, the mobile UI will shift this element to the top of its parent element. This is useful if, due to layout reasons, your main content is below other content in your HTML.
mobileUI-main-content-destination
Optionally used in conjunction with mobileUI-main-content. Instead of moving your main content to the top of its parent element, it'll be moved to the top of this element instead.

Custom Styling

If you don't want to use the built-in themes, here's how to add your own custom styling to the mobile UI:

  1. Set mobileUI_theme to none.

  2. Apply your own styles to the following in style-mobile.css:

    #mobileUI-site-titlebar
    The titlebar.
    #mobileUI-site-title
    The title.
    #mobileUI-site-nav-opener / #mobileUI-site-nav-opener:active
    The navigation panel's opener button (normal and active states).
    #mobileUI-site-nav
    The navigation panel itself.
    #mobileUI-site-nav-inner
    The navigation panel's inner element (wraps the links).
    .mobileUI-site-nav-link
    A navigation panel links.

  3. (Optional) If your styles change the width of the navigation panel opener, be sure to update mobileUI_openerWidth accordingly.

  4. You're done!

Settings

Using Settings

5grid uses an assortment of (purely optional) settings to dictate how it does its thing. Should the defaults not be to your liking, there are two ways you can put your own settings to use:

Edit 5grid/init.js
Pretty straightforward: open 5grid/init.js in a text editor, jump to the first section marked Settings and have at it. Just remember to copy these out if you upgrade 5grid down the road.
Add a query string to the <script> tag

You can add a standard query string to the 5grid/init.js <script> tag with as many settings as you like. For example:

<script src="5grid/init.js?use=mobile,1200px&mobileUI=1"></script>

The Settings

use
A comma-separated list of responsive modes to support (valid choices are any combination of mobile, desktop, 1000px, 1200px, and fluid; default is mobile,desktop). Specifying a single mode (eg. 1000px) will effectively disable 5grid's responsive handler, which can be pretty handy if you're not quite ready to jump into responsive design.
prefix
Stylesheet prefix (default is style). Used as the prefix for all of 5grid's stylesheets (eg. style.css, style-1000px.css, etc.)
breakpoint_1000px
Maximum screen width (in pixels) at which 1000px mode kicks in (default is 1200).
breakpoint_mobile
Maximum screen width (in pixels) at which mobile mode kicks in (default is 480).
viewport_is1000px
When in 1000px mode, page width to report to browsers that recognize the viewport meta tag (default is 1040).
viewport_is1200px
When in 1200px mode, page width to report to browsers that recognize the viewport meta tag (default is 1280).
mobileUI
Toggles the mobile UI on or off (can be 0 for disabled, 1 for enabled; default is 0).
mobileUI_hideAddressBar
(iOS only) Automatically hides the browser address bar on page load (can be 0 for disabled, 1 for enabled; default is 0).
mobileUI_openerText
Text to show on the opener (default is =).
mobileUI_openerWidth
Width of the mobile UI opener button in pixels (default is 60). This is used to determine the "gap" to leave on the right-hand side of the screen when the navigation panel is open.
mobileUI_slideSpeed
Speed (in ms) at which the navigation panel opens (can be any ms value or 0 for instant; default is 200).
mobileUI_theme
Selects the built-in theme to use (can be modern, beveled, flat, or none (if you want to completely style it yourself – see next section); default is modern).
mobileUI_themeNavColor
(when using a theme) Color of the mobile UI's navigation panel color (can be any valid color code; default is #272727).
mobileUI_themeTitleBarColor
(when using a theme) Color of the mobile UI's titlebar (can be any valid color code; default is #444444).
mobileUI_titleBarFixed
Determines if the mobile UI titlebar should be visible at all times (can be 0 for no, 1 for yes; default is 1).
mobileUI_titleBarHeight
Height of the mobile UI titlebar in pixels (default is 44).
mobileUI_titleBarOverlaid
Determines if the mobile UI titlebar should overlay the content (can be 0 for no, 1 for yes; default is 0).

Frequently Asked Questions

How do I test the other display modes?
Resize your browser window (either up or down), then refresh the page.
I'm testing my 5grid project in Internet Explorer and it's asking me to explicitly allow scripting – WTF? Are all my IE users going to have to deal with this?
Don't panic! IE's default security policy for files in the Local Machine Zone (ie. ones on your own hard drive) doesn't allow them to use scripting which, as you've already noticed, is needed by 5grid to work. However, since this only affects local files, the message won't appear to anyone viewing your stuff on an actual website (which, in IE terminology, falls under the more relaxed Internet Zone). So stay calm, click Allow, and carry on.
